# 変更案の具体化: 運用ルール（モデル更新時の同期）

目的: モデル更新時の齟齬を防ぎ、要件・設計・タスクの一貫性を維持する。

## 基本ルール

- 問題モデルは単一の真実として扱う（要件と設計の両方で同一モデルを参照）。
- モデル変更が入ったら、requirements/design/tasks の関連箇所を必ず同期更新する。

## 変更フロー（最小）

1. モデル差分の作成
   - 追加/削除/修正の一覧を明記する。
2. 要件の更新
   - 受け入れ基準と整合するように修正する。
3. 設計の更新
   - 制約の担保手段と実装箇所を更新する。
4. タスクの更新
   - モデル参照を修正し、差分を反映する。

## 変更のレビュー観点

- 差分が要件・設計・タスクの全てに反映されているか。
- 制約IDの変更があった場合、参照が途切れていないか。
- 状態変数の変更が、行動の前提/効果に反映されているか。

## 運用メモ

- モデル差分はチケット/PRに添付する。
- 変更時はvalidate-design/validate-implの再実行を推奨する。
